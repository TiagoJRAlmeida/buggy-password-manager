# buggy-password-manager

Project 2 for the class of Security in Software Engineering. It consists on correcting bugs and vunerabilities in a Password Manager App built with C. 


## Project structure

```
.
├── corrected_code/
├── original_vulnerable_code/
├── project_description/
├── docs/
└── README.md
```

The folder `original_vulnerable_code/` should not be touched. It's only purpose is to serve as comparison when needed.

The folder `corrected_code/` Is the one where we will be making the changes until the code is free from bugs. There is also a README.md file there.
It would be helpful if when correcting the errors we would write to that README.md so that it becomes easier to write the report later.

The folder `project_description/` just holds a copy of the site with the project description for easier offline work. To check it out all it's needed is to open the index.html file on a browser.

The folder `docs/` is currently empty, but should hold the different versions of the report.
